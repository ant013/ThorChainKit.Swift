Before every test, build, or smoke run, capture each root's `git rev-parse
HEAD`, `git status --porcelain=v1 --untracked-files=all`, `git diff --binary`,
`git diff --cached --binary`, and SHA-256 manifest of intentionally dirty or
untracked in-scope files. Verify the UW manifest's local `MarketKit.Swift` and
`ThorChainKit.Swift` paths resolve to the fixed roots. The ThorChainKit root must
equal the exact implementation PR head. The evidence is invalid if any other
captured SHA, dirty-input digest, status, or dependency resolution differs
between the reviewed baseline and the run.

## Step 1 — Correct the verified account-absence contract

**Owner:** ThorChainSwiftEngineer after explicit design approval.

**Dependencies:** This step starts only after CodeReviewer adversarial review
accepts the spec and the user explicitly approves the exact design revision.

**Affected paths:**

- `Sources/ThorChainKit/Network/LiveThorNodeClient.swift`
- `Tests/ThorChainKitTests/LiveThorNodeClientS1_04Tests.swift`

**Acceptance:** The account reader validates the leased height before any
absence classification, rejects duplicate JSON object keys, accepts both
verified address-specific absence messages with code `5` and empty details,
and keeps generic/foreign/malformed 404 bodies as typed account HTTP errors;
balance handling is unchanged.

**Verification:** Focused regression test fails before the fix for the short
body and for the height-before-classification invariant, then passes after it;
the exact negative matrix fails closed, including same-value and conflicting
duplicate keys; the full directly affected ThorChainKit test file passes;
balances 404 with the same body remains `.httpStatus(.balances, 404)`; diff
contains only the approved predicate/test delta.
