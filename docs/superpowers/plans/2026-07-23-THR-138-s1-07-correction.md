Before every test, build, and smoke run, call `Scripts/capture-s1-07-inputs.py` once per root, before and after the run. It emits only canonical digest JSON, never a patch or raw status. The schema is:

```json
{"schemaVersion":1,"rootLabel":"...","head":"<40-hex>","statusSha256":"<64-hex>","files":[{"path":"<JSON-escaped UTF-8 relative path>","state":"present|deleted|symlink-rejected","size":0,"sha256":"<64-hex>"}]}
```

The helper's exact rules are: file membership is the bytewise-UTF-8 sorted union of `git ls-files -c -o --exclude-standard -z`; tracked deletions remain in the manifest; regular files are hashed from working-tree bytes; symlinks fail closed; JSON uses `ensure_ascii=true`, sorted keys, compact separators, and one final newline; `statusSha256` hashes the raw NUL-delimited `git status --porcelain=v1 --untracked-files=all -z` bytes and does not retain them. It also hashes each resolved local `MarketKit.Swift` and `ThorChainKit.Swift` package path. ThorChainKit must equal the implementation PR head. Any before/after digest or package-resolution mismatch invalidates the run.
