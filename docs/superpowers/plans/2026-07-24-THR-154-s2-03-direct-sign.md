# THR-154 — S2-03 Local SIGN_MODE_DIRECT Codec

## Goal

Implement the internal, narrow Cosmos/THOR `MsgSend` `SIGN_MODE_DIRECT` codec
described by the canonical S2-03 spec. It produces deterministic `SignDoc`
bytes, a SHA-256 sign digest, signed `TxRaw`, and a local transaction hash.

## Scope boundary

In scope: pinned SwiftProtobuf-generated messages and provenance, one native
`/types.MsgSend`, exact direct-sign assembly, official and compatibility golden
vectors, compact-signature framing, independent verification of the static
golden fixture, and local hashing.

Out of scope: private-key or signer ownership, broadcast, pending state,
journaling, UI, host integration, generic Cosmos messages, `MsgDeposit`,
Maestro in Unstoppable, production signature-trust validation (including
wrong-key, invalid/high-S, and supplied-key verification), and S2-04/S2-05
work. S2-04 owns those signer-trust checks.

## Steps and acceptance checks

1. Add the pinned generated message subset, provenance manifest, and
   deterministic regeneration check.
   Check: `Scripts/generate-protobuf.sh --check` passes and the provenance test
   confirms pinned source, protoc, plugin, include roots, and checksums.

2. Add the internal codec for one `MsgSend`, fixed `rune`, empty fee coins,
   `SIGN_MODE_DIRECT`, gas `3_000_000`, exact memo, and exact `Data` reuse.
   Check: invalid addresses, key encodings, amounts, sequences, memos,
   signature framings, messages, signers, fees, type URLs, and alternate
   gas/sign modes fail closed without network activity or byte logging;
   cryptographic signer-trust checks are not implemented here.

3. Add complete official-gas, Vultisig compatibility, and scalar-one signed
   golden controls, including intermediate bytes and independent decode.
   Check: exact `SignDoc`, digest, signature bytes, `TxRaw`, and uppercase hash
   match; the static fixture's independent test-oracle verification and
   one-bit hash mutation checks pass.

4. Add public-surface and local verification guards.
   Check: generated/SwiftProtobuf types and sensitive bytes do not escape the
   internal boundary; focused tests, full local `swift test`, and `git diff
   --check` pass.

## Analog decision

Vultisig `THORChainHelper` is the primary THOR assembly and static
signature-verification-control spine. EvmKit supplies independent local exact-byte/hash evidence;
TronKit supplies SwiftProtobuf generation feasibility. Vultisig's 20M-gas
helper is a rejected counterexample. Production signer trust belongs to S2-04.
THORNode's pinned protocol constants and
the complete golden vectors remain authoritative.

## Analog delta matrix

| Preserved invariant | Required S2-03 delta | Proof |
|---|---|---|
| Vultisig fixes THOR assembly order and provides a verification control | Move that order into an internal pinned SwiftProtobuf codec with exact signature framing; keep public-key verification only as an independent static-fixture test oracle | Full intermediate-byte decode and signature golden |
| EvmKit hashes locally encoded bytes | Hash exact serialized `TxRaw` with SHA-256 and uppercase hex output | TxRaw fixture and one-bit mutation test |
| TronKit keeps generated sources inside the kit | Pin only required messages, generator inputs, versions, and checksums | Provenance no-diff and public-surface audit |
| THORNode defines consensus wire values | Fix `/types.MsgSend`, `rune`, empty fee coins, direct mode `1`, and gas `3_000_000` | Official-gas vector and field-by-field decode |
| Vultisig exposes a 20M-gas helper | Reject custom production gas; retain 20M only as isolated compatibility control | Product-source guard and compatibility vector |

Rejected deltas: generic Cosmos message support, signer/private-key ownership,
network calls, broadcast, pending behavior, host/UI composition, and S2-04+
lifecycle work.

## Test plan

Tests precede implementation and observe behavior, not generated implementation
details:

- complete official-gas and scalar-one signed vectors for every intermediate
  message, `SignDoc`, digest, signature framing, `TxRaw`, and local hash;
- isolated Vultisig compatibility digest using explicit legacy 20M gas;
- independent decode round trip for type URLs, one message/signer/signature,
  literal `rune`, empty fee coins, mode `1`, and gas `3_000_000`;
- deterministic repeated construction and one-bit `TxRaw` mutation;
- fail-closed address/key encodings, canonical numeric text and overflow, memo
  byte limits, signature framing lengths, extra messages/signers, nonempty
  fees, alternate mode, custom gas, and unknown type URLs;
- independent static-fixture signature verification as a test oracle; invalid,
  high-S, wrong-key, and supplied-key signer-trust behavior is covered by
  S2-04;
- public API/import audit for protobuf type and sensitive-byte leakage;
- provenance check followed by focused golden tests, public-surface tests,
  full local `swift test`, and `git diff --check`.
