# THR-135 — S2-01 Send Domain and Quote Contract

## Goal

Implement only the approved S2-01 public immutable send/quote domain and Kit
facade contract from architecture revision 10 at commit
`518835315a65996b9321665213adb0516503df65`. This plan is a new formalization
revision. The exact commit containing this revised plan and spec is the only
approval target; it is recorded in the Paperclip handoff and PR before approval.

The resulting package exposes reviewable native-RUNE values and a host-owned
signer capability without exposing private keys, raw transaction construction,
account/sequence/fee overrides, or upstream error data.

## Assumptions

- Sprint 1 `Kit`, `Address`, `Network`, lifecycle, Combine, and the package's
  declared BigInt `5.0.0` floor / committed default `5.7.0` lock remain
  authoritative; there is no BigInt `5.3.0` assumption.
- The iOS 13 library floor and UI-agnostic `Sources/ThorChainKit` boundary are
  unchanged.
- Implementation starts only after explicit approval of the final spec and
  this plan revision.
- Network preflight, protobuf, byte-level signing verification, journal,
  transport, durable pending/retry runtime, Example UI, host integration, and
  S2-02 remain out of scope. S2-01 owns public contracts and an in-memory
  contract fixture only.

## Acceptance criteria

The plan uses the spec's single canonical list: S2-01-A1 public graph,
S2-01-A2 immutable snapshots, S2-01-A3 facade/composition,
S2-01-A4 fail-closed deferred engines, S2-01-A5 quote lifecycle,
S2-01-A6 validation/errors, S2-01-A7 reflection/pending ordering, and
S2-01-A8 local verification. No second acceptance list is allowed.

## Analog conclusion

- Primary spine: current Unstoppable SendNew handler split (`sendData` for
  review/preflight and `send` for authorization), typed validation data, and
  ten-second expiration.
- Supporting roles: current Unstoppable `SendHandlerFactory` composition,
  current TronKit Kit/TransactionSender/TransactionManager ownership boundary
  and pending composition, current ThorChainKit Sprint 1 public API test seam,
  and a greenfield immutable-domain delta. TronKit raw transaction authority is
  a rejected delta, not the S2-01 primary.
- Rejected counterexample: EvmKit raw transaction plus signature submission and
  seed/private-key signer factories; those violate the host-neutral authority
  boundary.
- Required delta: move authoritative quote/review state into immutable
  ThorChainKit values with internal-only construction and an opaque one-use
  QuoteStore identity; add the stored `SendRuntime` dependency through
  `KitDependencies`/both `KitFactory` paths and bridge activation/invalidation
  from `LifecycleCommandBridge` at `LifecycleGate`. Preserve Kit ownership and
  explicit review/send split.

## Implementation steps

### 1. Public immutable domain graph

- Owner: ThorChainSwiftEngineer
- Files: `Sources/ThorChainKit/Send/Domain/*`, `Send/Signer.swift`,
  `Send/TransactionID.swift`, `Send/PendingTransaction.swift` as needed by the
  approved package layout.
- Tests first: public-consumer compile probe; strict-concurrency `Sendable`
  probes; value round-trip tests above and below `UInt64`; memberwise-
  initializer visibility checks; explicit redacted debug projections for
  `SendQuote`, `SigningRequest`, and `PendingTransaction`.
- Acceptance: criteria 1 and 3.

### 2. Kit facade, composition, and quote identity

- Owner: ThorChainSwiftEngineer
- Files: `Sources/ThorChainKit/Core/Kit+Send.swift`, `Core/Kit.swift`,
  `Core/KitDependencies.swift`, `Core/KitFactory.swift`,
  `Sync/LifecycleCommandBridge.swift`, and
  `Sources/ThorChainKit/Send/Internal/QuoteStore.swift`.
- Tests first: lifecycle precedence, stop/start generation invalidation, late
  H0 callback discard, ten-second expiry boundary, namespace binding, and
  sequential/concurrent one-use consumption. QuoteStore uses a 32-byte secure
  token, an injected monotonic clock, and an atomic consume linearization point.
  The facade snapshots optional retry fees before its first actor hop. Active
  calls fail with `operationUnavailable` and zero dependency-spy calls until
  later engines exist; pending replays `[]` with `.degraded` and no storage work.
- Acceptance: S2-01-A3, A4, and A5; no transport or signer work on rejected local
  input. S2-01 proves only the admission boundary; durable
  journal/recovery/operation-hold behavior is handed to S2-05.

### 3. Stable sanitized errors

- Owner: ThorChainSwiftEngineer
- Files: `Sources/ThorChainKit/Send/Errors/SendError.swift` and only directly
  required supporting value files.
- Tests first: every public case and payload; empty `QuoteChanges` rejection;
  deterministic sorted descriptions; fixed allowlisted broadcast codespace and
  diagnostic categories; credential/URL/key/signature/SignDoc/TxRaw/raw-
  codespace/response-body redaction canaries through debug and reflection paths.
- Acceptance: S2-01-A1, A6, and A7.

### 4. Exact-head review and independent QA

- Owner: ThorChainCodeReviewer, then ThorChainQAEngineer
- Scope: exact PR head only; review against the S2-01 spec and this plan.
- Verification applies to the post-implementation PR head: `swift package
  dump-package`; each named focused test filter; complete local `swift test`;
  `bash Scripts/verify-s2-01-public-surface.sh` with expected compile-negative
  probes; `bash Scripts/verify-s2-01-concurrency.sh --dependency 5.7.0` and
  `--dependency 5.0.0`; and `bash Scripts/verify-bigint-floor.sh`. Each focused
  filter must show a nonzero discovered test count. PR evidence must cite one
  exact head SHA for CR and QA, `git diff --name-only`, local logs,
  `gh pr checks <PR>` with every required check conclusion, the CR approval
  comment/review artifact, build-only GitHub Actions conclusions, import,
  secret, and reflection audits, and an explicit QA PASS. The current exact
  worktree already contains `Package.swift`; package commands apply to the
  implementation head, while GitHub Actions remain build-only.
- Acceptance: S2-01-A1 through A8 mapped to named tests/harnesses, no S2-02 or
  later-slice files, and no secrets or host/UI imports.

## Explicit non-goals

No preflight implementation, protobuf codec, byte-level signing verification,
journal, durable pending/retry runtime, transport, Example UI, host adapter,
Vultisig integration, or S2-02 work. `SigningRequestFactory` is a later-slice
byte-consistency responsibility; S2-01 only defines the immutable public
request shape and internal construction precondition.

codec/byte-consistency tests; S2-04 owns signer binding and signature
verification tests.
